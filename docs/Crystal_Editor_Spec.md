# Crystal Editor: 结晶知识编辑器深度设计规范 (X-Core Edition)

**核心使命**: 打造一个“低摩擦、高沉淀”的知识容器，支持从碎片 memo 到硬核文档的全形态记录。

---

## 1. 场景化设计 (Three Modes of Existence)

编辑器的底层虽然是统一的 Markdown 块架构，但在视觉和交互上支持三种模式的平滑切换：

### 1.1 **Record 模式 (记录流)** 
*   **场景**: 会议纪要、每日随笔、工作碎碎念。
*   **视觉**: 类似“朋友圈”或“垂直时间轴”，每条记录自带时间戳。
*   **痛点解决**: 解决“想写点东西但不知道放哪”的心理压力。

### 1.2 **Knowledge 模式 (知识页)**
*   **场景**: 技术文档、SOP 流程、项目深度分析。
*   **视觉**: 沉浸式单文档页面，呼吸感强，支持多级标题。
*   **痛点解决**: 提供深度的思考环境，支持双向链接 (`[[Link]]`) 织成的知识网。

### 1.3 **Memo 模式 (备忘卡)**
*   **场景**: 待办清单、临时灵感、紧急提醒。
*   **视觉**: 紧凑的卡片布局，支持任务勾选 (TODO) 和优先级标记。
*   **痛点解决**: 快速抓取、快速消化的临时信息。

---

## 2. 交互理念：Vibe-Driven Interaction

### 2.1 "/" 指令中心 (The Hub)
*   放弃所有顶部工具栏。输入 `/` 呼出命令面板。
*   支持 `/todo` (任务)、`/code` (代码块)、`/ai` (续写)、`/image` (图片)。

### 2.2 Ghost Writing (幻影补全)
*   **停顿即建议**: 当用户停顿 500ms 时，预测后续内容并以 20% 透明度显示。
*   **Tab 键采纳**: 极低的操作成本，让“写笔记”像“玩游戏”一样爽快。

### 2.3 虫洞穿梭 (Cross-linking)
*   **引流**: 聊天中的对话可以一键发送给编辑器。
*   **回溯**: 文档中的某段话可以带上下文链接到当时的聊天记录。

---

## 3. 视觉规范 (X-Core Aesthetics)

*   **色彩体系**: #0A0A0A (漆黑底) + #1A1A1A (碳纤维灰) + #00D1FF (工业蓝点缀)。
*   **字体选择**: 标题使用 `Inter`，正文使用具备机械感的 `Roboto Mono` 或自定义等宽字体。
*   **UI 元素**: 
    *   **0 滚角**: 拒绝圆润，保持硬朗线条。
    *   **动态背光**: 编辑器聚焦时，背景有极微弱的呼吸蓝光。
    *   **代码原生感**: 编辑器本身就像是一个高级 IDE 的精简版。

---

## 4. 技术实现路径 (The Engine)

1.  **内核**: 选用 Tiptap (基于 ProseMirror)，因为它支持完全的“去头化（Headless）”定制。
2.  **存储**: 采用 JSON-Document 存储而非纯 Markdown 字符串，以支持更复杂的元数据。
3.  **实时性**: 每一行文字的变动异步同步到后端 MySQL，无感保存。
